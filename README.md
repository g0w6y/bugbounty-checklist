# Bug Bounty Field Checklist

A professional, in depth bug bounty checklist for three target classes, presented as a clean interactive web page.

## Target classes

1. **Web Application.** Fifteen phases from scope and passive recon through JavaScript analysis, the login page, session and access control, injection, SSRF, business logic, and reporting.
2. **API.** Aligned to the OWASP API Security Top 10 with recon, object and function level authorisation, rate controls, injection, inventory, and GraphQL specifics.
3. **Open Source Code.** Source level review covering supply chain, secrets in history, language specific sinks, memory safety, and coordinated disclosure.

## Features

* Per item checkboxes with progress tracking saved in the browser.
* Live filter across every item.
* Collapsible phases and per domain completion counts.
* No build step. Static HTML, CSS, and JavaScript.

## Run locally

Open `index.html` in a browser, or serve the folder with any static server.

## Scope

For authorised security testing only. Stay in scope and disclose responsibly.
